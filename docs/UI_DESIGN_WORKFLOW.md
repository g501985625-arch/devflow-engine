# UI 设计工作流规范

## 阶段概述

UI 设计阶段（ui_design）是 DevFlow Engine 工作流中的第 3 个阶段，位于架构阶段之后、开发阶段之前。

## 工作流程

```
┌─────────────────────────────────────────────────────────────┐
│                    UI 设计阶段工作流                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 规划师 ──────────────────────────────────────────────▶ │
│     │  任务链规划                                            │
│     │  - 定义 UI 设计任务                                    │
│     │  - 规划设计优先级                                      │
│     │  - 分配任务给美术                                      │
│                                                             │
│  2. 美术 ─────────────────────────────────────────────────▶ │
│     │  前端 UI 设计                                          │
│     │  - 使用 Kimi K2.5 Visual Coding                       │
│     │  - 上传参考截图                                        │
│     │  - 生成 React + Tailwind 代码                         │
│     │  - 迭代调整配色/布局/动效                              │
│                                                             │
│  3. 主管 ─────────────────────────────────────────────────▶ │
│     │  验证                                                  │
│     │  - 视觉效果验证                                        │
│     │  - 交互体验测试                                        │
│     │  - 功能完整性检查                                      │
│                                                             │
│  4. 总管 ─────────────────────────────────────────────────▶ │
│     │  操作业                                                │
│     │  - 协调资源                                            │
│     │  - 处理阻塞问题                                        │
│     │  - 推进任务进度                                        │
│                                                             │
│  5. 主管 ─────────────────────────────────────────────────▶ │
│     │  视觉操作验证                                          │
│     │  - 最终视觉确认                                        │
│     │  - 用户体验测试                                        │
│     │  - 批准进入下一阶段                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 角色职责

### 美术 (Designer)

| 职责 | 说明 | 工具 |
|------|------|------|
| 前端 UI 设计 | 使用 Kimi K2.5 Visual Coding 生成 UI | kimi.com/websites |
| 截图分析 | 上传参考截图，分析设计风格 | Kimi Vision |
| 代码生成 | 输出 React + TailwindCSS 代码 | Visual Coding |
| 迭代调整 | 根据反馈调整配色、布局、动效 | Kimi 对话 |
| 设计规范 | 输出配色方案、组件规范文档 | Markdown |

### 主管 (Supervisor)

| 职责 | 说明 | 验证标准 |
|------|------|----------|
| 视觉效果验证 | 检查 UI 渲染效果 | 颜色/布局/字体符合设计规范 |
| 交互体验测试 | 测试 Hover/Click/动画效果 | 动效流畅，交互响应正常 |
| 功能完整性检查 | 确认组件功能完整 | 所有组件可正常使用 |
| 视觉操作验证 | 最终视觉确认并批准 | 符合用户体验标准 |

### 规划师 (Planner)

| 职责 | 说明 | 输出 |
|------|------|------|
| 任务链规划 | 规划 UI 设计任务顺序 | 任务链配置 |
| 优先级排序 | 确定设计任务优先级 | 优先级列表 |
| 任务分配 | 分配任务给美术 | 任务分配表 |

### 总管 (Manager)

| 职责 | 说明 | 输出 |
|------|------|------|
| 操作业执行 | 执行具体操作任务 | 操作记录 |
| 资源协调 | 协调设计资源 | 资源分配 |
| 问题处理 | 处理阻塞和异常 | 问题解决方案 |

## 任务类型

### UI 设计任务

| 任务类型 | 描述 | 执行者 |
|----------|------|--------|
| `ui_design_reference` | 收集参考设计截图 | 美术 |

| `ui_interaction_test` | 交互体验测试 | 主管 |
| `ui_final_approve` | 最终视觉批准 | 主管 |

### 规划任务

| 任务类型 | 描述 | 执行者 |
|----------|------|--------|
| `ui_task_plan` | 规划 UI 设计任务链 | 规划师 |
| `ui_priority_assign` | 分配任务优先级 | 规划师 |

### 操作任务

| 任务类型 | 描述 | 执行者 |
|----------|------|--------|
| `ui_resource_coord` | 协调设计资源 | 总管 |
| `ui_issue_resolve` | 处理阻塞问题 | 总管 |

## Kimi K2.5 Visual Coding 使用规范

### 输入要求

美术在使用 Kimi Visual Coding 时需提供：

1. **参考截图**: Linear、Dribbble、Behance 等优秀设计截图
2. **设计描述**: 风格、配色、布局要求
3. **技术栈**: React + TailwindCSS + TypeScript
4. **组件列表**: 需要生成的 UI 组件

### Prompt 模板

```markdown
请使用 React + TailwindCSS + TypeScript 生成以下 UI：

参考风格: [上传截图]

技术要求:
- 框架: React 19
- 样式: TailwindCSS 4
- 字体: Inter
- 图标: Lucide Icons

配色方案:
- 背景: #0F172A (Slate-900)
- 卡片: #1E293B (Slate-800)
- 主色: #6366F1 (Indigo)
- 成功: #10B981 (Emerald)

组件列表:
1. [组件名称] - [功能描述]
2. [组件名称] - [功能描述]
...
```

### 输出要求

美术需输出以下内容：

1. **React 组件代码**: 完整的组件实现
2. **样式配置**: TailwindCSS 配置
3. **设计规范文档**: 配色、字体、间距规范
4. **使用说明**: 组件使用方法

## 验证标准

### 视觉效果验证

| 检查项 | 标准 |
|--------|------|
| 颜色一致性 | 符合设计规范配色 |
| 布局合理性 | 间距、对齐符合规范 |
| 字体正确性 | 使用指定字体 |
| 图标正确性 | 使用指定图标库 |

### 交互体验测试

| 检查项 | 标准 |
|--------|------|
| Hover 效果 | 动效流畅，颜色变化正确 |
| Click 效果 | 响应正常，状态切换正确 |
| 动画效果 | 过渡流畅，无卡顿 |
| 加载状态 | Loading 动效正常 |

### 功能完整性检查

| 检查项 | 标准 |
|--------|------|
| 组件可用 | 所有组件可正常渲染 |
| 数据绑定 | Props/State 正常传递 |
| 事件处理 | onClick/onHover 正常响应 |
| 错误处理 | 错误状态正常显示 |

## 与其他阶段的关系

```
架构阶段 (architecture)
    │
    │  输出: 系统架构设计、模块划分
    │
    ▼
UI 设计阶段 (ui_design)
    │
    │  输入: 架构设计中的前端模块需求
    │  输出: UI 组件代码、设计规范文档
    │
    ▼
开发阶段 (development)
    │
    │  输入: UI 组件代码
    │  任务: 接 API + WebSocket + 功能实现
    │
    ▼
...
```

## 异常处理

| 异常类型 | 处理方式 | 处理者 |
|----------|----------|--------|
| 设计不符合规范 | 反馈给美术迭代调整 | 主管 |
| Kimi 生成失败 | 使用备用方案或人工设计 | 总管 |
| 视觉验证失败 | 反馈具体问题，要求修正 | 主管 |
| 任务阻塞 | 协调资源，解决问题 | 总管 |
